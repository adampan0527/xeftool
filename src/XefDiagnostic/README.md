# XEF 文件诊断工具

用于验证 XEF 文件完整性和诊断读取问题的命令行工具。

## 功能

- ✓ 验证 XEF 文件头是否有效
- ✓ 统计各类型帧的数量（彩色、深度、红外）
- ✓ 定位文件损坏的具体位置
- ✓ 输出详细的诊断报告
- ✓ 提供问题原因分析和建议

## 编译

```bash
cd src/XefDiagnostic
dotnet build -c Release
```

编译后的可执行文件位于：
`src/XefDiagnostic/bin/Release/net10.0/XefDiagnostic.dll`

## 使用方法

### 诊断单个文件

```bash
dotnet run --project src/XefDiagnostic -- "C:\path\to\file.xef"
```

或者使用编译后的 DLL：

```bash
dotnet src/XefDiagnostic/bin/Release/net10.0/XefDiagnostic.dll "C:\path\to\file.xef"
```

### 批量诊断多个文件

```bash
dotnet run --project src/XefDiagnostic -- file1.xef file2.xef file3.xef
```

### 诊断示例文件

```bash
dotnet run --project src/XefDiagnostic -- "XEF2JPEG_Input\chen\20260514_034238_00.xef" "XEF2JPEG_Input\chen\20260514_043701_00.xef" "XEF2JPEG_Input\chen\20260514_045922_00.xef"
```

## 输出说明

工具会输出以下诊断信息：

### 阶段1：文件头验证
- 文件签名（应该是 "EVENTS1"）
- 报告的流数量

### 阶段2：流定义扫描
- 发现的所有流类型（彩色、深度、红外等）
- 是否压缩

### 阶段3：逐帧读取
- 每100帧输出一次进度
- 错误发生位置（如果有）
- 扫描恢复尝试（如果有）

### 阶段4：诊断汇总
- 帧数统计
- 数据读取百分比
- 错误恢复统计
- 性能统计
- **诊断结论和建议**

## 诊断结论解读

### ✓ 文件读取完成，数据基本完整
- 读取进度 > 90%
- 文件基本正常，可能有少量错误但已恢复

### ⚠ 警告: 文件读取提前终止
- 读取进度 < 90%
- 文件存在损坏区域
- 会显示最后有效位置的地址

### ⚠ 警告: 文件读取在非常早期就终止了
- 读取进度 < 1%
- 文件严重损坏或格式错误
- 需要重新获取文件

## 技术说明

该工具使用与 XefTool 相同的解析逻辑，但增加了详细的诊断输出：
- 每个错误的精确位置（文件偏移）
- 扫描恢复的详细过程
- 最后有效读取的位置
- 数据读取的百分比

## 故障排除

### 问题：文件读取在非常早期就终止
**原因**：文件头部数据损坏或格式错误
**解决**：
1. 使用十六进制编辑器（如 HxD、010 Editor）检查文件开头
2. 验证文件签名是否为 "EVENTS1"
3. 检查文件是否完整（没有被截断）

### 问题：文件读取提前终止（进度 < 90%）
**原因**：文件中间存在大面积损坏（>1MB）
**解决**：
1. 查看诊断报告中的"最后有效位置"
2. 使用十六进制编辑器检查该位置之后的数据
3. 考虑从备份恢复或重新录制

### 问题：扫描恢复失败
**原因**：损坏区域超过 1MB，无法自动恢复
**解决**：
1. 这表明文件存在严重的连续损坏
2. 需要重新获取文件或从备份恢复
